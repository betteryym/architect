## 案例分析考点详解

### 架构案例分析真题分类

根据历年真题考点分析，架构案例分析真题可分为以下几大类：

1. **软件架构设计**：每年必考1-2题，第1题必选题，必须掌握。主要涉及质量属性、软件架构风格、软件架构评估、MVC架构、面向服务的架构SOA、ESB、J2EE架构等。对于其他未考察的架构领域重点知识如DSSA、ABSD等，也必须重点掌握。

2. **系统开发基础**：几乎每年必考1题。主要涉及UML的图、关系的识别，尤其是类图、用例图、活动图、状态图；设计模式识别；数据流图、E-R图等简单识别；信息安全相关技术；项目管理-进度管理-关键路径。

3. **数据库系统**：偶尔会考察一题。主要考察数据库新技术的比较，如关系型数据库、内存数据库及NoSQL等，还包括反规范化技术、主从复制、负载均衡等。

4. **嵌入式系统**：几乎每年必考一题，选做题。考察较多的是嵌入式系统的实时性和可靠性以及容错等概念。大概率会考察一些嵌入式领域陌生技术，如果是完全没见过的技术，不选即可。

5. **Web应用开发**：主要考察Web相关技术，一般结合架构进行考察。偶尔会考察新技术，遇到完全没听说过的技术，不选即可。
